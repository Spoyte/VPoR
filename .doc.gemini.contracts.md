# VPoR Smart Contract Specifications

## 1. Overview
The VPoR system relies on two main contracts:
1.  `GlassVaultEVVM.sol`: The core logic contract deployed on the VPoR Chain. It inherits from `Evvm.sol` to leverage Async Nonces and stores the liability root and verified asset total.
2.  `ChainlinkEVVMBridge.sol`: A bridge contract deployed on the Host Chain (Sepolia) that fetches data from Chainlink Functions and forwards it to `GlassVaultEVVM`.

## 2. GlassVaultEVVM.sol
**Role:** The Single Source of Truth for Solvency (EVVM Extension).

### Inheritance
-   Inherits from `Evvm.sol` (MATE Stack).

### State Variables
```solidity
// The Merkle Root of the current liability set
bytes32 public liabilityRoot;

// The total sum of liabilities (from the Merkle Sum Tree)
uint256 public totalLiabilities;

// The total assets verified by Chainlink
uint256 public totalAssets;

// Timestamp of the last liability update
uint256 public lastLiabilityUpdate;

// Timestamp of the last asset update
uint256 public lastAssetUpdate;

// Whitelisted Liability Publishers (Exchange Backend)
mapping(address => bool) public isLiabilityPublisher;

// The Bridge address on Sepolia (for asset updates)
address public chainlinkBridge;
```

### Events
```solidity
event LiabilitySnapshot(bytes32 indexed root, uint256 totalLiabilities, uint256 timestamp);
event AssetVerification(uint256 totalAssets, string assetType, uint256 timestamp);
event ProofOfInclusion(address indexed user, bytes encryptedPayload);
```

### Functions
#### `submitLiabilityProof(bytes32 _root, uint256 _totalLiabilities, address[] calldata _users, bytes[] calldata _encryptedProofs)`
-   **Access:** Restricted to Liability Publisher.
-   **Logic:**
    1.  Updates `liabilityRoot` and `totalLiabilities`.
    2.  Updates `lastLiabilityUpdate`.
    3.  Emits `LiabilitySnapshot`.
    4.  Iterates through `_encryptedProofs` and emits `ProofOfInclusion` for each user.

#### `updateVerifiedAssets(uint256 _totalAssets, string memory _assetType)`
-   **Access:** Restricted to Chainlink Bridge.
-   **Logic:**
    1.  Updates `totalAssets`.
    2.  Updates `lastAssetUpdate`.
    3.  Emits `AssetVerification`.

#### `isSolvent()`
-   **Type:** View.
-   **Logic:** Returns `totalAssets >= totalLiabilities`.

#### `getReserveRatio()`
-   **Type:** View.
-   **Logic:** Returns `(totalAssets * 10000) / totalLiabilities` (Basis Points).

## 3. ChainlinkEVVMBridge.sol
**Role:** The Bridge (Sepolia).

### Logic
This contract receives the data from the Chainlink DON and calls `GlassVaultEVVM`.

#### `fulfillRequest(bytes32 requestId, bytes memory response, bytes memory err)`
-   **Role:** Callback from Chainlink Functions.
-   **Logic:**
    1.  Decodes `response` (uint256 assetBalance).
    2.  Calls `GlassVaultEVVM.updateVerifiedAssets(assetBalance, "BTC")`.

## 4. Security Considerations
-   **Async Nonces:** `GlassVaultEVVM` leverages the underlying EVVM architecture to process liability updates in parallel.
-   **Access Control:** Strict separation of concerns between Liability Publisher (Exchange) and Asset Publisher (Chainlink).
-   **Data Availability:** The `encryptedProofs` are emitted as events on the VPoR Chain.
