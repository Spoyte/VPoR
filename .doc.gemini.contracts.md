# VPoR Smart Contract Specifications

## 1. Overview
The VPoR system relies on two main contracts:
1.  `GlassVault.sol`: The core logic contract deployed on the VPoR Chain. It stores the liability root and the verified asset total.
2.  `ChainlinkEVVMAdapter.sol`: A bridge contract deployed on the Host Chain (Sepolia) or the VPoR Chain (depending on the exact EVVM bridge architecture) to translate Chainlink responses into VPoR updates.

## 2. GlassVault.sol
**Role:** The Single Source of Truth for Solvency.

### State Variables
```solidity
// The Merkle Root of the current liability set
bytes32 public liabilityRoot;

// The total sum of liabilities (from the Merkle Sum Tree)
uint256 public totalLiabilities;

// The total assets verified by Chainlink
uint256 public totalAssets;

// Timestamp of the last liability update
uint256 public lastLiabilityUpdate;

// Timestamp of the last asset update
uint256 public lastAssetUpdate;

// Whitelisted executors
mapping(address => bool) public isExecutor;
```

### Events
```solidity
event LiabilitySnapshot(bytes32 indexed root, uint256 totalLiabilities, uint256 timestamp);
event AssetVerification(uint256 totalAssets, string assetType, uint256 timestamp);
event ProofOfInclusion(address indexed user, bytes encryptedPayload);
```

### Functions
#### `submitLiabilityProof(bytes32 _root, uint256 _totalLiabilities, bytes calldata _encryptedProofs)`
-   **Access:** Restricted to Exchange Backend (Whitelisted).
-   **Logic:**
    1.  Updates `liabilityRoot` and `totalLiabilities`.
    2.  Updates `lastLiabilityUpdate`.
    3.  Emits `LiabilitySnapshot`.
    4.  Iterates through `_encryptedProofs` and emits `ProofOfInclusion` for each user (or batches them).

#### `updateVerifiedAssets(uint256 _totalAssets, string memory _assetType)`
-   **Access:** Restricted to Chainlink Adapter (Whitelisted).
-   **Logic:**
    1.  Updates `totalAssets`.
    2.  Updates `lastAssetUpdate`.
    3.  Emits `AssetVerification`.

#### `isSolvent()`
-   **Type:** View.
-   **Logic:** Returns `totalAssets >= totalLiabilities`.

#### `getReserveRatio()`
-   **Type:** View.
-   **Logic:** Returns `(totalAssets * 10000) / totalLiabilities` (Basis Points).

## 3. ChainlinkEVVMAdapter.sol
**Role:** The Translator.

### Logic
This contract receives the data from the Chainlink DON and formats it for the `GlassVault` contract.

#### `fulfillRequest(bytes32 requestId, bytes memory response, bytes memory err)`
-   **Role:** Callback from Chainlink Functions.
-   **Logic:**
    1.  Decodes `response` (uint256 assetBalance).
    2.  Calls `GlassVault.updateVerifiedAssets(assetBalance, "BTC")`.
    *Note: If the Adapter is on Sepolia and GlassVault is on the VPoR Chain, this function must trigger a cross-chain message or an EVVM transaction.*

## 4. Security Considerations
-   **Replay Attacks:** All updates must include a timestamp or nonce to prevent replaying old states.
-   **Access Control:** The `onlyExecutor` modifier is critical. The whitelist must be managed by a secure Admin (Multisig).
-   **Data Availability:** The `encryptedProofs` are emitted as events. We rely on the EVVM chain's data availability for users to retrieve their proofs.
