# **VPoR Implementation Plan (Permissioned Chain Edition)**

## **Phase 1: The Logic (Safe Mode) \- Hours 0-8**

**Goal:** Get the GlassVault contract logic working on the public MATE Testnet first. This ensures you have a working demo even if the custom chain setup takes too long.

* \[ \] **Contract Development (GlassVault.sol)**  
  * \[ \] Implement submitLiabilityProof(): Updates Merkle Root & Total Liabilities (Admin Only).  
  * \[ \] Implement updateVerifiedAssets(): Updates Total Assets (Oracle Only).  
  * \[ \] Implement notifyUsers(): Emits encrypted UserData events for privacy.  
  * \[ \] Add isSolvent() view function.  
* \[ \] **Testing**  
  * \[ \] Write Foundry tests to verify the Merkle Sum Logic and Solvency Math.  
  * \[ \] **Deploy to MATE Testnet** (Sepolia) as a backup.

## **Phase 2: The Sovereign Chain (Infrastructure) \- Hours 8-16**

**Goal:** Deploy your own EVVM instance where *only* you and Chainlink can write transactions.

* \[ \] **EVVM Core Setup**  
  * \[ \] Clone evvm-org/Testnet-Contracts.  
  * \[ \] Configuration: Enable **Permissioned Executors** (Restrict executeTransaction to whitelist).  
  * \[ \] Token: Set to "Gasless" (or Native VPoR Token) since you are subsidizing the chain.  
* \[ \] **Deploy EVVM Core to Sepolia**  
  * \[ \] Run the deployment script.  
  * \[ \] **Result:** You get an EvvmCoreAddress on Sepolia. This IS your Virtual Chain.  
* \[ \] **Access Control**  
  * \[ \] **Whitelist Executor 1:** Your Backend Wallet (for Liability Updates).  
  * \[ \] **Whitelist Executor 2:** *Reserved for Chainlink Adapter (Phase 3\)*.  
* \[ \] **Deploy GlassVault to VPoR Chain**  
  * \[ \] Target your new Custom RPC.  
  * \[ \] Deploy GlassVault.sol.

## **Phase 3: The Chainlink Executor Bridge \- Hours 16-24**

**Goal:** Allow Chainlink (on Sepolia) to execute transactions on your VPoR Chain (Virtual).

* \[ \] **The Adapter Contract (ChainlinkEVVMAdapter.sol)**  
  * \[ \] *Location:* Deploy this on **Sepolia** (Host Chain).  
  * \[ \] *Function:* performUpkeep or fulfillRequest.  
  * \[ \] *Logic:*  
    1. Receives Asset Data from Chainlink Functions.  
    2. Encodes a transaction call for GlassVault.updateVerifiedAssets().  
    3. Calls EvvmCore.executeTransaction(payload) on Sepolia.  
* \[ \] **Permissions**  
  * \[ \] **Whitelist Executor 2:** Add the address of ChainlinkEVVMAdapter to your EVVM Core whitelist.  
  * \[ \] *Result:* Now Chainlink can write to your Virtual Chain, but random users cannot.  
* \[ \] **Chainlink Functions Setup**  
  * \[ \] Create Subscription.  
  * \[ \] Deploy the verify\_assets.js script.  
  * \[ \] Connect the Consumer to your Adapter contract.

## **Phase 4: The Trusted Fisher (Backend) \- Hours 24-30**

**Goal:** Automate the Liability Updates from your "Exchange".

* \[ \] **Liability Generator Script**  
  * \[ \] Create dummy user data (JSON).  
  * \[ \] Generate Merkle Sum Tree (Root \+ Total Sum).  
  * \[ \] Generate ECIES Encrypted blobs for each user.  
* \[ \] **The Executor Script (Node.js)**  
  * \[ \] Use ethers.js with your **Whitelisted Backend Wallet**.  
  * \[ \] Construct the transaction: GlassVault.submitLiabilityProof(...).  
  * \[ \] Send it to the **EVVM Core** (on Sepolia) or via your Custom RPC.  
  * \[ \] *Outcome:* The Virtual Chain state updates.

## **Phase 5: Frontend & User Verification \- Hours 30-36**

**Goal:** A clean UI for users to decrypt their data.

* \[ \] **Scaffold**  
  * \[ \] Next.js \+ RainbowKit \+ Wagmi.  
  * \[ \] Configure Custom Network (VPoR Chain) in Wagmi config.  
* \[ \] **Public Dashboard**  
  * \[ \] Display Total Assets vs Total Liabilities.  
  * \[ \] Show "Solvency Ratio" (Green/Red).  
* \[ \] **Private Dashboard**  
  * \[ \] **Decrypt Button:** Triggers window.ethereum.request({ method: 'eth\_decrypt' }).  
  * \[ \] **Verify:** Show "Your Balance: 5 ETH (Included in Tree ✅)".  
* \[ \] **Demo Prep**  
  * \[ \] Show a failed transaction from a non-whitelisted wallet (proving the Permissioned Security).
