# VPoR Chainlink Integration Strategy

## 1. The Challenge: Functions vs. CRE
The ETHGlobal Buenos Aires prize track for Chainlink emphasizes the **Chainlink Runtime Environment (CRE)**. However, `gemini_doc.md` was designed around **Chainlink Functions**.

### Strategy
1.  **Primary Implementation (Functions):** We will build the system using Chainlink Functions first because it is well-documented and we have a clear path to execution.
2.  **Prize Upgrade (CRE):** Once the Functions implementation is working, we will attempt to wrap the logic in a CRE Workflow if the documentation allows.

## 2. Chainlink Functions Implementation

### The Source Script (`source.js`)
This JavaScript code runs on the Chainlink DON.
```javascript
// Example logic for fetching BTC balance
const address = args[0];
const url = `https://api.blockcypher.com/v1/btc/main/addrs/${address}/balance`;

const request = Functions.makeHttpRequest({
  url: url,
});

const response = await request;

if (response.error) {
  throw Error("Request failed");
}

const balance = response.data.final_balance; // Satoshis
return Functions.encodeUint256(balance);
```

### The Consumer Contract
The `ChainlinkEVVMAdapter` acts as the Consumer.
-   **Subscription ID:** We need to create a subscription on the Chainlink Functions UI.
-   **DonHostedSecrets:** API keys for Blockcypher/Infura should be encrypted and hosted by the DON.

## 3. Chainlink CRE Implementation (Target)
If we migrate to CRE, the architecture changes slightly:

### The Workflow
Instead of a simple Request/Response, we define a **Workflow**.
1.  **Trigger:** Cron Schedule (e.g., every 1 hour) OR On-Chain Event.
2.  **Action:** Fetch Data (using the same logic as `source.js`).
3.  **Consensus:** The DON agrees on the result.
4.  **Target:** Execute the transaction on the VPoR Chain.

### Advantages for Prize
-   **"Connect the World":** CRE is the new standard for connecting off-chain data.
-   **Orchestration:** We can chain multiple checks (e.g., Check BTC Balance AND Check Bank API) into a single atomic workflow.
